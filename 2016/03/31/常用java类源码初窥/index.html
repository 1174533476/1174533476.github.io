1.List系列

List系列常用的实现类是LinkedList和ArrayList

ArrayList

内部使用一个数组来实现List

private transient Object[] elementData;

Java的serialization提供了一种持久化对象实例的机制。当持久化对象时，可能有一个特殊的对象数据成员，我们不想
用serialization机制来保存它。为了在一个特定对象的一个域上关闭serialization，可以在这个域前加上关键字transient。
transient是Java语言的关键字，用来表示一个域不是该对象串行化的一部分。当一个对象被串行化的时候，transient型变量的值不包括在串行化的表示中，然而非transient型的变量是被包括进去的
转自:http://www.blogjava.net/fhtdy2004/archive/2009/06/20/286112.html

ArrayList中插入一个元素需要将此元素后面的都往后移动,

public void add(int index, E element) {
rangeCheckForAdd(index);

    ensureCapacityInternal(size + 1);  // Increments modCount!!
    System.arraycopy(elementData, index, elementData, index + 1,
                     size - index);
    elementData[index] = element;
    size++;
}

关于System.arraycopy() http://xuyuanshuaaa.iteye.com/blog/1046621

ArrayList中删除元素时,并不会将内部的elementData变小

LinkedList

内部使用内部对象存储,
public class LinkedList
extends AbstractSequentialList
implements List, Deque, Cloneable, java.io.Serializable

如果有大量随机访问对象的需求,就使用ArrayList,如果有大量插入删除使用LinkedList

2.Map系列

几个常用Map实现类:HashMap TreeMap LinkedHashMap ConcurrentHashMap

HashMap

使用数组和链表实现,在HashMap中使用一个数组,内部的hash()计算出hash值一样的数放入同一个Entry中,Entry中使用链表存储

 transient Entry<K,V>[] table = (Entry<K,V>[]) EMPTY_TABLE;
 static class Entry<K,V> implements Map.Entry<K,V> {
    final K key;
    V value;
    Entry<K,V> next;
    int hash;
}

HashMap其实就是一个Entry数组，Entry对象中包含了键和值，其中next也是一个Entry对象，它就是用来处理hash冲突的，形成一个链表。

关于HashMap很贴切的解释 http://www.cnblogs.com/ITtangtang/p/3948406.html 包含loadFactor加载因子的内容

LinkedHashMap

LinkedHashMap继承自HashMap,在HashMap基础上面增加了TreeMap的顺序输出功能,

// Entry中,增加了next
static class Entry<K,V> implements Map.Entry<K,V> {
    final K key;
    V value;
    Entry<K,V> next;
    int hash;
}
public V put(K key, V value) {
    if (table == EMPTY_TABLE) {
        inflateTable(threshold);
    }
    if (key == null)
        return putForNullKey(value);
    int hash = hash(key);
    int i = indexFor(hash, table.length);
    for (Entry<K,V> e = table[i]; e != null; e = e.next) {
        Object k;
        if (e.hash == hash && ((k = e.key) == key || key.equals(k))) {
            V oldValue = e.value;
            e.value = value;
            e.recordAccess(this);//<----------
            return oldValue;
        }
    }

    modCount++;
    addEntry(hash, key, value, i);
    return null;
}

ConcurrentHashMap

这是一个线程安全的HashMap,在Collection中也提供了线程安全类
public static Map synchronizedMap(Map m) {
return new SynchronizedMap(m);
}
SynchronizedMap中所有的方法都加上了synchronized,保证了方法的同步,但在使用中仍然会有安全问题
ConcurrentHashMap中使用了,锁分段技术具体细节刨个坑,以后再填

http://blog.csdn.net/madun/article/details/6326337

关于HashMap和hashTable

HashMap中key和value都可以是null,而在hashtable中key和value都不能是null
hashtable是线程安全的,但是效率低下,全局使用了一把锁,当其他线程访问时,容易堵塞
